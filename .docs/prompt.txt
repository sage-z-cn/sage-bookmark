为名为"sage bookmark"的浏览器插件编写详细的开发设计文档，该插件需遵循以下技术规范和功能要求：

### 技术规范
1. 开发框架：采用Manifest V3标准进行开发
2. 前端框架：使用React框架开发界面组件，采用函数式组件和Hooks模式
3. UI界面：使用 Ant Design 进行界面开发，遵循 genesis-DESIGN.md，将 genesis-DESIGN.md 的设计变量映射到 Ant Design 的 Design Token
4. 状态管理：使用React Context API或Redux等状态管理方案管理应用状态
5. 浏览器兼容性：确保完全兼容最新版Microsoft Edge和Google Chrome浏览器
6. 代码规范：遵循现代JavaScript/TypeScript开发标准，确保代码可维护性和扩展性
7. 构建工具：使用Vite或Webpack等现代构建工具进行项目构建

### 核心功能需求
开发一个浏览器收藏夹管理插件，主要功能包括：
- 提供整页式选项页面，实现类似Windows资源管理器的收藏夹管理界面
- 支持两种视图模式切换：大图标模式和详细列表模式
- 实现完整的收藏夹项目管理功能：移动、排序、新增、删除操作
- 支持多选操作功能，可对多个项目同时执行操作
- 实现拖拽功能，允许用户通过拖拽方式移动收藏夹项目
- 支持双击操作打开收藏夹项目或文件夹

### 整页选项页面设计规范
页面布局分为四个主要部分：

1. **顶部地址栏**
   - 功能按钮：返回、前进、刷新
   - 路径显示：当前浏览的收藏夹路径，格式类似Windows资源管理器地址栏
   - 搜索功能：在当前路径下搜索收藏夹项目，支持模糊匹配

2. **顶部工具栏**
   - 操作按钮：新建(文件夹/书签)、剪切、复制、重命名、分享、删除
   - 视图切换：提供大图标模式/详细列表模式切换按钮

3. **主体内容区域**
   - 显示当前路径下的收藏夹项目
   - 支持项目排序（名称、日期、类型等）
   - 实现项目选择功能（单选/多选）
   - 支持拖拽操作实现项目移动
   - 支持框选

4. **底部状态栏**
   - 显示统计信息：当前文件夹的项目总数
   - 显示选择状态：已选中的项目数量

### 特殊技术要求
1. 实现高效的收藏夹数据处理，确保在大量收藏项目下仍保持良好性能
2. 实现收藏夹数据的本地存储与同步机制
3. 确保操作的安全性，关键操作需提供确认机制
4. 实现响应式设计，适配不同屏幕尺寸
5. 提供适当的错误处理和用户提示机制
